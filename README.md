# koe
STEP2課題アプリ

## 開発環境
- Python 3.11
- streamlit
- Supabase

## 初期セットアップ

### 1. リポジトリを取得
source treeでこのリポジトリをclone

### 2. 仮想環境の作成

ここからターミナル（コマンドプロンプト）で実行

- cloneしたフォルダへ移動
cd koe

- 仮想環境の作成
Mac: python3 -m venv venv
Windows: python -m venv venv

- 仮想環境を有効化
Mac: source venv/bin/activate
Windows: venv\Scripts\activate

- Streamlit アプリを起動する
streamlit run app.py

- ライブラリのインストール
pip install -r requirements.txt

- 仮想環境の終了
deactivate

## ライブラリを途中で追加する場合

仮想環境にいることを確認し、ターミナルで実行
pip install ライブラリ名

requirements.txtに追加したライブラリ名を追記してpush
注意：バージョン情報は不要
- requests

他のメンバーは追加されているライブラリをローカル上でインストール
pip install ライブラリ名


# 🎯 プロジェクトの本質的な問い

> 「探しても見つからないのは検索エンジンの問題ではなく、データが存在しないことが問題。Koeはそのデータを、AIの時代に備えて今から積み上げるインフラ。」
> 

黒崎CDOが解決したい「社員が『探す』ことに費やす無益なコスト」の根本は、**ドキュメント化されていない1万2千人の『声』が存在しないこと**にある。

## 💡 コアコンセプト

**これまでドキュメント化されていなかった現場の「声」をデータ化し、中間管理職が即座にアクセスできるようにする。**

- 宝の山は「声」
- AIが参照しやすい形で蓄積すれば、AIが進化するほど価値が増す
- 創業70年・1万2千人の暗黙知が特に重要な資産

## コンセプト詳細

## 本質的な課題：黒崎CDOへのカウンターロジック

黒崎氏が掲げる「社員が『探す』という無益な時間に費やすコスト」をゼロにするためには、検索機能の強化だけでは不十分です。

- **真のボトルネック**: 検索して見つからない最大の理由は、12,000人の従業員が持つ知見がドキュメント（文字）になる前に消えている」ことにあります。
- **データの空白**: 創業70年で積み上げられた「現場のコツ」「違和感」「熟練の技」は、誰のPCにも、どのフォルダにも存在しません。
- **Koeの提供価値**: 検索エンジンを作るのではなく、**AIが検索・活用するための「生データ」を生成し続けるインフラ**を構築します。

---

## 資産化すべき「声」の定義

経営上の重要度（インパクト）が高い情報を「4つの資産」として定義します。

| **カテゴリー** | **具体的な「声」の例** | **ビジネスインパクト（経営価値）** |
| --- | --- | --- |
| **熟練の暗黙知** | 「テレビを叩いたら直る」「JAF並みに鍵開けが上手い人がいる」 | **技術継承・属人化解消**：高齢化による後継者不足対策。 |
| **リスクの芽** | 「今日は腰が痛い」「ラインBで変な音がした（ヒヤリハット）」 | **損失回避**：労働災害や設備故障の未然防止によるコスト削減。 |
| **ビジネスの種** | 「この端材、別の製品に使えそう」「現場のこの工程が非効率」 | **新規事業・変革**：現場発のアイデアをドキュメント化し、事業化へ。 |
| **称賛・他薦** | 「〇〇さんのあの対応が神がかっていた」 | **組織活性化**：隠れたキーマンの可視化とモチベーション向上。 |

---

## AI時代の「競争優位性」

AIの性能が汎用化するこれからの時代、企業の強さは「AIの賢さ」ではなく「AIに読み込ませる独自データの質と量」で決まる。

- **社内独自情報の蓄積**: 外部のAIが持っていない「テクゼロン固有の現場知」が溜まるほど、AIの回答精度は上がり、競合が真似できない強みになる。
- **ビジネスチャンスの創出**: 企業が新しい課題に直面した際、データベース化された過去の「声」をAIが横断的に検索することで、即座に解決策やヒントを提示できるようになる。

---

## 形式知化の戦略：なぜ「音声日報」か？

## 1. 「書く」から「話す」への転換：抽出コストの最小化

熟練工の頭の中にある知恵を引き出す際、従来の「書く」「聞く」には限界がありました。

- **「書く」の限界**: 現場のプロはPC入力が苦手なことが多く、言語化する過程で「重要なコツ」が面倒な事務作業として削ぎ落とされる。
- **「聞く（インタビュー）」の限界**: 専門のインタビュアーが必要で、12,000人の現場をカバーするのはコスト的に不可能。
- **音声の優位性**: 現場でスマホに**「独り言」を吹き込むだけ**で完了。一人で完結し、記録時間はタイピングの数分の一で済みます。

## 2. データの質の変化：ニュアンス（暗黙知）の保存

ドキュメント化されると消えてしまう「感覚的な情報」こそが、実は最も価値のあるデータです。

- **生きた情報の回収**: 「いつもより少し振動が重い」「なんとなく嫌な予感がする」といった、**書くほどではないが重要な違和感**は、音声だからこそ残ります。
- **コンテキスト（文脈）の保持**: 声のトーンやスピードから、現場の緊急度や疲労度までがデータに含まれ、AIが「行間」を読み取るための強力な手がかりになります。

## 3. 日報という「接触頻度」：資産化のパイプライン

特別な「ナレッジ共有会議」を設けるのではなく、日常の「日報」に組み込むことが重要です。

- **非同期のインタビュー化**: 日報を「今日やったことの報告」ではなく、**「AIによる非同期インタビュー」**と再定義。
- **圧倒的な蓄積スピード**: 毎日、作業直後の新鮮な記憶が吹き込まれるため、数ヶ月で「70年分のマニュアル」を凌駕する生きたデータベースが構築されます。

---

### **Phase 1：習慣化（今週のMVP）**

- **ゴール**: 音声入力による蓄積に慣れてもらう。
- **主要項目**: 「気になったこと」「あの人のここがすごい」など、入力の心理的ハードルを下げる項目の実装。

### **Phase 2：資産化（将来展望）**

- **ゴール**: ベテランの暗黙知をAIで構造化。
- **主要項目**: 蓄積されたデータをAIが解析し、「〇〇について知っている人」を特定したり、設備故障の予兆を検知したりする。

## 🗺️ フェーズ計画

| フェーズ | 内容 |
| --- | --- |
| **MVP① ← 現在の焦点** | 中間管理職のための日報データベース（入力→データ化→整理→通知） |
| **MVP②** | ベテラン技術者の暗黙知の形式知化 |
- 2段階実装戦略詳細：習慣化から資産化へ
    
    黒崎CDOへは、「まずは日報で現場のパイプラインをつくり、次に本丸である熟練工の知恵を抜き出す」という2段構えで説明します。
    
    ---
    
    ## 🟦 フェーズ1：音声日報による「データの習慣化」
    
    **【狙い】現場の声を漏らさず吸い上げる「インフラ」の構築**
    
    - **ターゲット**: 現場の非デスクワーカー（広範囲）
    - **解決する課題**:
        - 報告が「事務作業（書くこと）」になっており、生きた情報が削ぎ落とされている。
        - 「探しても見つからない」以前に、現場の一次データがそもそも存在しない。
    - **入力内容**:
        - 今日の進捗、トラブルの有無、体調、ちょっとした気づき。
    - **生み出す価値**:
        - **リスクの早期発見**: 「なんか変だ」という声をリアルタイムで管理職がキャッチ。
        - **データ生成**: AIが活用するための「生データ」が毎日自動で蓄積される状態を作る。
    
    > **💡 ポイント**
    > 
    > 
    > まずは「スマホに向かって話せば、自分の声が届く」という体験を全社に浸透させ、データが止まらずに流れる「土壌」を作ります。
    > 
    
    ---
    
    ## 🟧 フェーズ2：熟練者の「暗黙知の形式知化」
    
    **【狙い】失われていく「職人の知恵」をデジタル資産（護城河）に変える**
    
    - **ターゲット**: ベテラン技術員・特定のスペシャリスト（重要層）
    - **解決する課題**:
        - 70年培った「職人のコツ」や「判断基準」が言語化されず、退職と共に消滅している。
        - インタビュー等の従来手法ではコストが高すぎて、彼らの知見を網羅できない。
    - **入力内容**:
        - 「この機械のこの音は、ここが原因だ」「気温が高い日はこの設定にしろ」といった、マニュアルにない秘伝のタレ。
    - **生み出す価値**:
        - **技術継承の自動化**: 熟練者がいなくなっても、AIがその「声」を元に後継者をガイドできる。
        - **圧倒的な差別化（Moat）**: 競合他社が逆立ちしても手に入らない「自社固有の成功パターン」がデータベース化される。
    
    > **💡 ポイント**
    > 
    > 
    > フェーズ1で作った「音声入力の土壌」を活用し、最も価値の高い「ベテランの頭の中」を効率的に、かつ網羅的に吸い出します。
    > 
    
    ---
    
    ## 🔄 フェーズ間のつながり（なぜこの順番か）
    
    1. **信頼と慣れ**: いきなり「秘伝の技を教えろ」と言っても職人は動きません。まずは日報という日常業務を楽にすること（Phase 1）でツールへの信頼を得ます。
    2. **AIの学習**: 日報（Phase 1）で溜まった大量の文脈があるからこそ、AIは熟練者の高度な一言（Phase 2）の重要性を正しく理解し、構造化できるようになります。
    
    ---
    
    ### 💡 黒崎CDOへの「殺し文句」
    
    > 「黒崎さん、我々は2つのステップでテクゼロンを変えます。
    > 
    > 1. **Phase 1**で、12,000人の日常から『データの空白』を埋めます。
    > 2. **Phase 2**で、最も貴重な『職人の頭の中』を資産化し、他社が真似できないAIインフラを完成させます。
    > 
    > これこそが、失われた10年を取り戻し、次の70年を支える唯一の道です。」

## 🏗️ アーキテクチャ（3層構造）

### フロントエンド

入力UI（現場作業者向け）

- 徹底的にシンプルなUI
- スマホの音声入力→テキスト化→送信
- 入力項目：業務内容 / 気になったこと / あの人のここがすごい（他薦）

検索・通知UI（管理職向け）

- ログイン時に「今日対応すべき問題」だけがポンと出てくる
- 緊急度・上長判断の要否でフィルタリング
- リスク（体調不良・不満・危険）を自動検出・通知
